# 🤖 AI-ассистент обработки отзывов

Система автоматической обработки отзывов клиентов с анализом тональности и генерацией AI-ответов.

## 📋 Описание проекта

Проект состоит из двух связанных компонентов:

1. **app_test_2803** — веб-приложение для сбора и отображения отзывов
2. **worker_ai** — автономный сервис для анализа и обработки отзывов

## ✨ Возможности

- Приём отзывов через веб-интерфейс
- Автоматическое определение тональности (positive/negative/neutral)
- Генерация AI-ответов через OpenAI API
- Отправка уведомлений о негативных отзывах в Telegram
- Поддержка вложенных комментариев (ответы на отзывы)
- Защита от повторной обработки и дублирования

## 🏗️ Архитектура системы

```
Пользователь → app_test_2803 (веб-приложение)
                    ↓
            PostgreSQL (хранение отзывов)
                    ↓
            worker_ai (опрос каждые 10 сек)
                    ↓
    ┌───────────────┼───────────────┐
    ↓               ↓               ↓
OpenAI API    Telegram Bot    AI-ответ в приложение
(анализ)      (уведомления)   (комментарий)
```

### Взаимодействие компонентов

**app_test_2803** (`app_test_2803/`):
- `main.py` — точка входа FastAPI приложения
- `api/routes.py` — API endpoints для работы с отзывами
- `models/review.py` — модель данных отзыва
- `db/` — подключение к PostgreSQL через SQLAlchemy

**worker_ai** (`worker_ai/`):
- `worker.py` — главный цикл обработки отзывов
- `client.py` — HTTP-клиент для работы с API приложения
- `processor.py` — анализ тональности и генерация ответов (OpenAI + fallback)
- `telegram_bot.py` — отправка уведомлений в Telegram
- `state.py` — локальное состояние обработанных отзывов

### Файл состояния (`data/state.json`)

Хранит ID обработанных отзывов и отправленных уведомлений. При каждом запуске worker проверяет:
- Если отзыв уже в state → пропускает обработку
- Если нет → обрабатывает и добавляет в state

Это предотвращает:
- Дублирование AI-ответов при перезапуске
- Повторные уведомления в Telegram
- Бесконечные цепочки автоответов

## 🚀 Быстрый старт

### Требования
- Docker и Docker Compose
- OpenAI API ключ (опционально, без него работает fallback-логика)
- Telegram Bot Token (опционально, для уведомлений)

### Установка и запуск

1. **Клонирование репозитория:**
```bash
git clone https://github.com/holmok1577-ops/parser_i_rabota_s_otzivami.git
cd parser_i_rabota_s_otzivami
```

2. **Настройка переменных окружения:**
```bash
# Для веб-приложения
cp app_test_2803/.env.example app_test_2803/.env

# Для worker'а
cp worker_ai/.env.example worker_ai/.env
```

Отредактируйте `.env` файлы:

**app_test_2803/.env:**
```env
POSTGRES_DB=reviews_db
POSTGRES_USER=reviews_user
POSTGRES_PASSWORD=reviews_password
WORKER_API_TOKEN=your-secret-token
```

**worker_ai/.env:**
```env
OPENAI_API_KEY=sk-your-openai-key
TELEGRAM_BOT_TOKEN=your-telegram-bot-token
TELEGRAM_CHAT_ID=your-chat-id
WORKER_API_TOKEN=your-secret-token  # Должен совпадать с app
```

3. **Запуск через Docker Compose:**
```bash
# Сначала веб-приложение с базой данных
cd app_test_2803
docker-compose up --build -d

# Затем worker
cd ../worker_ai
docker-compose up --build -d
```

4. **Проверка работы:**
- Веб-приложение: http://localhost:8000
- API отзывов: http://localhost:8000/api/reviews

## 🔧 Конфигурация

### Переменные окружения

**Общие:**
| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `WORKER_API_TOKEN` | Токен для авторизации worker | `change-me` |

**Worker:**
| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `OPENAI_API_KEY` | API ключ OpenAI | пусто (fallback) |
| `OPENAI_MODEL` | Модель GPT | `gpt-4.1-mini` |
| `WORKER_POLL_INTERVAL` | Интервал опроса (сек) | `10` |
| `TELEGRAM_BOT_TOKEN` | Токен бота | пусто |
| `TELEGRAM_CHAT_ID` | ID чата для уведомлений | пусто |

### Режимы работы

**С OpenAI API (полный функционал):**
- Точное определение тональности с учётом контекста
- Умная генерация персонализированных ответов
- Распознавание сарказма и эмоций

**Без OpenAI API (fallback-режим):**
- Keyword-based анализ тональности с контекстными исключениями
- Шаблонные ответы в зависимости от тональности
- Работает полностью оффлайн

## 📊 Путь данных (Data Flow)

1. **Появление отзыва:**
   - Пользователь создаёт отзыв через веб-интерфейс
   - Отзыв сохраняется в PostgreSQL со статусом `new`

2. **Обнаружение и анализ:**
   - `worker.py` каждые 10 секунд опрашивает API через `client.py`
   - Получает новые отзывы со статусом `new`
   - `processor.py` определяет тональность (OpenAI или fallback)

3. **Уведомление (только negative):**
   - Если тональность negative → `telegram_bot.py` отправляет уведомление
   - ID отзыва сохраняется в state для предотвращения повторов

4. **Генерация и публикация ответа:**
   - `processor.py` генерирует текст ответа
   - `client.py` создаёт комментарий-ответ к отзыву
   - Исходный отзыв помечается статусом `processed`

- Все токены и ключи вынесены в `.env` файлы
- `.env` добавлен в `.gitignore`
- Worker требует `WORKER_API_TOKEN` для API-запросов
- Защита от CSRF через заголовок `X-Worker-Token`

## 🆘 Устранение неполадок

**Worker не подключается к приложению:**
```bash
# Проверьте сеть
docker network ls
docker network inspect app_test_2803_default
```

**Повторная обработка отзывов:**
```bash
# Очистить state
rm worker_ai/data/state.json
docker restart reviews-worker
```

**Telegram-уведомления не приходят:**
- Проверьте `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`
- Убедитесь что бот может писать пользователю (нажмите /start в чате с ботом)
